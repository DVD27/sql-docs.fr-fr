# <a name="installing-ssma-components-on-sql-server-db2tosql"></a>Installation des composants SSMA sur SQL Server (DB2ToSQL)
Dans cette version de SSMA, il n’est pas nécessaire d’installer séparément [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] sur le pack d’extension SSMA, qui prend en charge la migration des données et les fournisseurs DB2 pour activer la connectivité de serveur à serveur.  
  
## <a name="ssma-for-db2-extension-pack"></a>SSMA pour DB2 Pack d’extension  
Le pack d’extension SSMA ajoute un schéma à la base de données dans [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]l’instance spécifiée de. Le schéma **sysdb** contient les tables et les procédures stockées nécessaires à la migration des données, ainsi que les fonctions définies par l’utilisateur qui émulent les fonctions système DB2.
