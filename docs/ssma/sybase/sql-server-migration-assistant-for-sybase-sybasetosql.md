# <a name="sql-server-migration-assistant-for-sybase-sybasetosql"></a>Assistant Migration SQL Server pour Sybase (SybaseToSQL)

[!INCLUDE[msCoName](../../includes/msconame_md.md)][!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Assistant Migration (SSMA) pour Sybase Adaptive Server Enterprise (ASE) est un outil de migration des bases de données [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] ASE vers [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 2012, [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 2014, [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] 2016, 2017 sur Windows et [!INCLUDE[msCoName](../../includes/msconame_md.md)] [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] Linux, 2019 sur Windows et Linux [!INCLUDE[msCoName](../../includes/msconame_md.md)] , ou Azure SQL Database. SSMA pour Sybase convertit les objets de [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] base de données ASE en objets de [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] base de données, crée ces objets dans ou Azure SQL Database, [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] puis migre les données de ASE vers ou Azure SQL Database.
  
Cette documentation vous présente SSMA pour Sybase et fournit des instructions pas à pas pour migrer des bases de données ASE [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] vers ou Azure SQL Database et des informations sur les problèmes qui peuvent se produire après la migration. Pour plus d’informations, consultez les articles suivants.
