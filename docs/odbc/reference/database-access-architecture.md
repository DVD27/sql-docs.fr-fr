# <a name="database-access-architecture"></a>Architecture de l’accès aux bases de données
L’une des questions soulevées dans le développement de l’ODBC était de savoir quelle partie de l’architecture d’accès à la base de données pour normaliser. Les interfaces de programmation SQL décrites dans la section précédente - les modules SQL intégrés, SQL et CLIs - ne sont qu’une partie de cette architecture. En fait, étant donné qu’ODBC était principalement destinée à connecter des applications personnelles par ordinateur à des DBMS mini-ordinateurs et à des cadres principaux, il y avait aussi un certain nombre de composants réseau, dont certains pouvaient être normalisés.
